- Parsing is the process of determing how a string of terminals can be generated from a grammar

- BNF https://en.wikipedia.org/wiki/Backus%E2%80%93Naur_form
  - Authors: John Backus, Peter Naur
  - metasyntax notation
  - can be used to express context-free grammars
  - parts
    - terminals
    - nonterminals: in <between>
    - productions:
      #+begin_src bnf
        <digit>   ::= 0 | 1 | 2 | 3 | 4 | 5
        <integer> ::= <digit> | <digit><integer>
        <float>   ::= <integer>.<integer>
      #+end_src
- EBNF https://en.wikipedia.org/wiki/Extended_Backus%E2%80%93Naur_form
  - +compact +complex

- (chomsky) hierachy of languages https://en.wikipedia.org/wiki/Chomsky_hierarchy
  - from + restrictive to - restrictive
    - non effectively computable
    - type 0 - recursive enumerable (recognized turing machines)
    - type 0 - recursive
    - type 1 - context sensitive
    - type 2 - context free / non deterministic
    - type 2 - context free / deterministic
    - type 3 - regular (fsa)

- finite automata, finite state machine (fsm)
  - "a kind of conceptual computer"
  - types:
    - DFA (deterministic)
      - each state can only go to another state given a particular input
    - NDFA/NFA (non-deterministic)
      - eg: (ha+)+h
  - parts
    - states (nodes)
    - transitions (vertex)
