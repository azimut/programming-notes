** BenEater: https://eater.net/vga
*** 1) intro [[https://www.youtube.com/watch?v=l7rce6IQDWs][The world's worst video card?]]
     - horizontal sync
       - 9 bit counter (3x 4bit)
       - using logic gates to detect timing events
         - 200, 210, 242, 264
         - by counter value
         - each, NAND'ing 8 raw/NOT bits known to be "1"
       - using an inverted SR-latch, to detect timing ranges
         - using 2 nand gates
     - vertical sync
       - same setup
       - 10 bit counter (3x 4bit)
       - 600, 601, 605, 628
*** 2) build [[https://www.youtube.com/watch?v=uqY3FMuMuRo][World's worst video card? The exciting conclusion]]
     - using an EEPROM
     - using the counters to tell where are we painting (eeprom address)
       - slowing vertical counter to keep a 200x200 instead of 200x600
       - dropping some bits due address limit on eeprom 100x75
     - using the eeprom data to give a color to each RGB component
       - using 2 bits for each
       - 1.5k + 680 resistors in parallel
       - 4 possible values of each
       - 64 colors
     - not sending anything for blank/control lines
